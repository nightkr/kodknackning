@()(implicit req: RequestHeader) @contentBase("Hej, världen!") {

Mitt första kommandoanrop

Nu när du har installerat Scala så kan du öppna Scalas "REPL" (Read-Eval-Print-Loop, vi återkommer till detta senare). Där kan du prova olika Scala-kommandon som sedan körs.

På Windows så kan det göras genom att trycka Win+R, skriva cmd och trycka Enter, och slutligen skriva scala och trycka på Enter.

Du kan köra ett kommando genom att skriva in det, och sedan trycka på Enter.

Ett vanligt första kommando som används i olika programmeringsguider brukar kallas för "Hello World", då det skriver ut den textsträngen i terminalen. Scalas variant står härunder, pröva den nu!

@helpers.sample( """println("Hej, Världen!")""")

Detta betyder att vi anropar println med indatat (brukar kallas för argumentet) "Hej, Världen!". Citationstecknen angränsar rå text som inte ska tolkas speciellt (textsträngar) och paranteserna angränsar kommandot från argumenten. println skriver ut sitt argument i kommandotolken och returnerar sedan ingenting, men andra kommandon kan göra vad som helst.

Matte

Vi kan även beräkna matematiska uttryck. Skriv helt enkelt in dem som vanligt.

@helpers.sample("println(3 * 5)")

Här kanske du också märkte att vi inte hade några citationstecken. Detta är för att siffror, till skillnad från vanlig text, måste tolkas närmre för att kunna användas för något som faktiskt drar nytta ifrån egenskapen att de inte bara är text.

Variabler

Vi kan även spara ett värde (såsom ett matematiskt uttryck, en textsträng, eller resultatet från ett kommandoanrop) i en variabel. Detta gör att vi senare kan använda det i ett annat kommandoanrop.

@helpers.sample( """val värdet = 5 |println(värdet)""".stripMargin)

Det nya här är val, som betyder att vi skapar en ny variabel, och dels = som betyder att vi sätter värdet på variabeln till uttrycket till höger.

Automatiska variabler

I REPL-en så blir fristående värden automatiskt variabler med genererade namn som börjar på res.

@helpers.sample( """"ett" |"två" |"tre"""".stripMargin )

Detta fungerar endast i REPL-en, inte i riktiga Scala-program.

}
